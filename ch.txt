PHASE 2 — APP CREATION & DATA MODELING
============================================================
Book ref: "Creating an application", "Creating the blog data models" and
          all its subsections

Tasks:
[ ] `python manage.py startapp catalog`
[ ] Inspect generated admin.py / apps.py / migrations/ / models.py /
    tests.py / views.py

Build the Book model incrementally, exactly following the chapter's build
order (do not skip straight to the final version — each sub-step is its own
checkpoint):

  a) Base fields (Ref: "Creating the Post model")
     title    -> CharField(max_length=250)
     slug     -> SlugField(max_length=250)
     synopsis -> TextField()      [maps to Post.body]
     __str__  -> return self.title

  b) Datetime fields (Ref: "Adding datetime fields")
     publish  -> DateTimeField(default=timezone.now)
                 [the date/time this catalog entry becomes publicly
                 visible — an editorial/CMS date, not the book's real
                 print date]
     created  -> DateTimeField(auto_now_add=True)
     updated  -> DateTimeField(auto_now=True)
     (optional aside, per the book: try db_default=Now() on `publish`
     instead of default=timezone.now to see the DB-computed default)

  c) Default sort order (Ref: "Defining a default sort order")
     class Meta: ordering = ['-publish']

  d) Database index (Ref: "Adding a database index")
     class Meta: indexes = [models.Index(fields=['-publish'])]

  e) Activate the app (Ref: "Activating the application")
     Add 'catalog.apps.CatalogConfig' to INSTALLED_APPS in
     bookshelf/settings.py

  f) Status field (Ref: "Adding a status field")
     class Status(models.TextChoices):
         DRAFT = 'DF', 'Draft'
         PUBLISHED = 'PB', 'Published'
     status -> CharField(max_length=2, choices=Status, default=Status.DRAFT)

  g) Many-to-one relationship (Ref: "Adding a many-to-one relationship")
     added_by -> ForeignKey(settings.AUTH_USER_MODEL, on_delete=CASCADE,
                             related_name='books_added')
     [represents the cataloger/librarian account responsible for this
     record — intentionally kept many-to-one to mirror Post.author.
     A real-world "book has many authors" relationship is many-to-many
     and is out of scope for Chapter 1]

Deliverable: catalog/models.py with the fully assembled Book model
Definition of done: model imports cleanly, `python manage.py check` passes.

============================================================

Post model             -> Book model

Post.publish            -> Book.publish
